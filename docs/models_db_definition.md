# This file includes fields of each model, their description, and whether they are a required field.
fields are listed in A-Z.
# User
id (Automatic) \
`User ID which is automatically generated by rails.` \
college (Required) \
`User's college, used for recommendation.` \
created_at (Automatic) \
`Account creation date which is automatically generated by rails.` \
cuhk_id (Required) \
`User's CUHK ID, we can either extract it from the input email or have the user input it.` \
email (Required) \
`User's email, used mainly for verification.` \
hostel (Optional) \
`User's hostel, used for further recommendation.` \
is_admin (Required) \
`checks if the user is an admin.` \
name (Required) \
`User's username.` \
password_digest (Required) \
`User's password.` \
profile_picture (Optional) \
`have to think of a default pfp if no picture is uploaded.` \
seller_rating (Optional) \
`Implementation is optional, self explanatory.` \
seller_review_count (Optional) \
`Implementation is optional, also self explanatory.` \
updated_at (Optional) \
`probably wouldn't be used, but can be used to store the last time a user has changed their info.` \
verification_otp (Required) \
`used for user verification.` \
verification_sent_at (Required) \
`time when user sent a verification request.` \
verified_at (Required) \
`time when the user have verified the account.` 

# Products
id (Automatic) \
`Product ID which is generated by rails.` \
buyer_id (Required if status = sold, FK to Users) \
`The buyer's user ID (foreign key to Users), can be empty if the status is 'Available' or 'Reserved'.` \
category_id (Required) \
`Refer to the category table.` \
contact (Required) \
`Contact information of the seller.` \
created_at (Automatic) \
`Product listing time which is generated by rails.` \
description (Optional) \
`A longer description of the product, not a necessary input.` \
image (Optional) \
`An user uploaded image of the product, not necessary for the user to upload.` \
location (Required) \
`The location to sell the item, defaults to the college/hostel the user belongs to.` \
name (Required) \
`The name of the item sold.` \
price (Required) \
`The price of the item.` \
seller_id (Required, FK to Users) \
`The seller's user ID.` \
updated_at (Automatic) \
`An update time automatically generated by rails.` 

 # Categories
 This table is filled by us only, not to be entered by user. \
 id (Automatic) \
 `An automatic id generated by rails.` \
category_name (manually input) \
`A list of categories allowed on the platform, manually input by us.` 

# Interests
List of interested users \
id (Automatic) \
`An automatic key generated by rails.` \
created_at (Automatic) \
`An automatically generated time by rails.` \
interested_id (Required, FK to Users) \
`User that is interested to a certain item.` \
product_id (Required, FK to Products) \
`The product which is interested by the user.` \
updated_at (Automatic) \
`An automatically generated time by rails.` 

# Price_Histories
shows the price history of an item \
id (Automatic) \
`An automatic key generated by rails.` \
date (Required) \
`The date and time which the price point happens.` \
price (Required) \
`The price at that date.` \
product_id (Required, FK to Products) \
`The product in question for the price history.` 

# Chats
Chat stores the different chats only, and not the message contained in each chat. \
id (Automatic) \
`An automatic key generated by rails.` \
created_at (Automatic) \
`Creation time generated by rails.` \
interested_id (Required) \
`One of the two participants in the chat, which is the user interested in the product.` \
item_id (Required) \
`The product the two users are discussing over.` \
seller_id (Required) \
`The other participant in the chat, which is the seller of the product.` \
updated_at (Automatic) \
`Update time generated by rails.` 

# Messages
id (Automatic) \
`An automatic key generated by rails.` \
chat_id (Required, FK to Chats) \
`Identify which chat it belongs to.` \
created_at (Automatic) \
`The time the message is sent.` \
message (Required) \
`The chat message.` \
sender_id (Required, FK to Users) \
`The person who sent the message.` \
updated_at (Automatic) \
`An automatic time generated by rails, probably not needed.` 

# CommunityItems
A forum-like advertisement board for college members. \
id (Automatic) \
`An automatic key generated by rails.` \
user_id (Required, FK to Users) \
`The user who posted the advertisement (must be the seller).` \
product_id (Required, FK to Products) \
`The product being promoted.` \
description (Required) \
`A catchy advertisement description for the college community.` \
college (Required) \
`The specific CUHK college this promotion is targeted towards.` \
created_at (Automatic) \
`The time the promotion was posted.` \
updated_at (Automatic) \
`The time the promotion was last updated.`
